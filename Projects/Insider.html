INSIDER - PREDICTING CHURN VISITORS¶

Introduction¶

Insider is collecting clickstream data from hundreds of different e-commerce websites. These raw events are continuously being processed to obtain insights about the visitors of our partners.

We provide a lot of different ways to personalize the user experience for our partners. One of them is targeting different user groups for different personalization campaigns using our AI-backed Predictive User Segments such as Likelihood to Purchase, Uninstall Prediction, Discount Affinity etc. .

In this problem, you are asked to model a segment called Likelihood to Churn which predicts whether or not a user will not visit the website again within the next 7 days, as this is the business metric for the data we are providing.

Problem Description¶

You are provided with raw clickstream data of an e-commerce website between 1 January 2020 and 27 February 2020 containing around 1.350.000 click events as training dataset.

A clickstream dataset means that each row corresponds to a page view event for an unique user and session. A page view event can be of several types such as cart page view, product page view, success (which means purchase) etc. Each row contains a user id and a session id. User id uniquely identifies a user across the dataset, whereas session id uniquely identifies a session across the dataset.

A session is a group of user interactions with the website that take place within a given time frame, in our case 30 minutes. For example a single session can contain multiple page views, events, social interactions, and ecommerce transactions. See Figure 1 below for an example session with multiple events.

A single user can open multiple sessions. Those sessions can occur on the same day, or over several days, weeks, or months. As soon as one session ends, there is then an opportunity to start a new session. A session ends either after 30 minutes of inactivity or at midnight.

As you are provided with raw clickstream data and your objective is to predict the inverse possibility of a new session in 7 days for each session, first you need to aggregate raw click data to end up with session data so that each row corresponds to a single session. Then you need to label your data with respect to the given objective above. After that you need to train and optimize a predictive model using your favorite framework. Finally, using the trained model you need to predict the possibility of not having a new session within 7 days for each session in the test dataset.

Generating Predictions¶

Test dataset is in the very same format of the train dataset. So you need to apply all the data processing techniques you applied to obtain your training dataset so that you can run your model on the test dataset.

The test dataset contains around 178.000 clickstream events for March, however, we have pruned the dataset so that it only contains the very first session of each user. There are around 35.000 unique sessions. You need to generate a prediction for every single session.

class Solution(object):
    def dayOfYear(self, date):
        days = [0,31,28,31,30,31,30,31,31,30,31,30,31]
        d = list(map(int,date.split("-")))
        if d[0] % 400 == 0:
            days[2]+=1
        elif d[0]%4 == 0 and d[0]%100!=0:
            days[2]+=1
        for i in range(1,len(days)):
            days[i]+=days[i-1]
        return days[d[1]-1]+d[2]
